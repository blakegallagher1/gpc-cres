# Frontend Design Skill

Build distinctive, production-grade interfaces for Entitlement OS. Every UI should look like it was designed by a human with taste ‚Äî not generated by AI.

## Stack

- **Framework:** Next.js 14 App Router (`apps/web/app/`)
- **Components:** shadcn/ui (`apps/web/components/ui/`)
- **Styling:** Tailwind CSS with CSS variables for theming
- **Icons:** Lucide React (`lucide-react`)
- **Animation:** Framer Motion (if in dependencies, check `package.json`)
- **Charts:** Recharts (if in dependencies)
- **State:** React hooks + server components where possible

## Before Writing Code

1. **Check existing components:**
   ```bash
   ls apps/web/components/ui/
   ls apps/web/components/
   ```
   Reuse what exists. Don't create a new Button when shadcn has one.

2. **Check existing pages for patterns:**
   ```bash
   ls apps/web/app/deals/
   ls apps/web/app/dashboard/
   ```
   Match the layout structure, nav patterns, and data fetching approach.

3. **Check the Tailwind config:**
   ```bash
   cat apps/web/tailwind.config.ts
   ```
   Use existing color tokens, spacing scale, and breakpoints.

## Design Principles

### Typography
- Clear heading hierarchy: `text-2xl font-semibold` ‚Üí `text-lg font-medium` ‚Üí `text-sm text-muted-foreground`
- Data-dense layouts for CRE professionals ‚Äî they want information, not whitespace
- Monospace for financial figures: `font-mono tabular-nums`

### Color
- Use semantic CSS variables: `bg-background`, `text-foreground`, `border-border`
- Financial metrics: `text-emerald-600` (positive), `text-red-600` (negative)
- Status badges: match existing pattern in `apps/web/components/`
- Dark mode: works automatically via CSS variables ‚Äî never hardcode colors

### Layout
- Responsive grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3`
- Card-based layouts for deal/property data
- Sticky headers for long scrollable content
- Sidebar nav follows existing app shell pattern

### Interaction
- Subtle hover states: `hover:bg-accent transition-colors`
- Loading skeletons, not spinners (use shadcn Skeleton)
- Optimistic UI updates where possible
- Toast notifications via shadcn `useToast`

## Anti-Patterns ‚Äî DO NOT

- No gradient backgrounds or glassmorphism
- No excessive rounded corners (`rounded-lg` max, not `rounded-3xl`)
- No placeholder images or lorem ipsum ‚Äî use real field names
- No inline styles ‚Äî Tailwind only
- No new UI primitives when shadcn/ui has one
- No `className` string concatenation ‚Äî use `cn()` from `@/lib/utils`
- No hardcoded colors ‚Äî use CSS variables
- No giant padding/margins ‚Äî CRE users want data density
- No decorative elements that don't serve a purpose

## Component Patterns

### Data table
```tsx
import { Table, TableHeader, TableRow, TableHead, TableBody, TableCell } from "@/components/ui/table"
```

### Form
```tsx
import { Input } from "@/components/ui/input"
import { Button } from "@/components/ui/button"
import { Label } from "@/components/ui/label"
```

### Card layout
```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent } from "@/components/ui/card"
```

### Dialog / Sheet
```tsx
import { Dialog, DialogContent, DialogHeader, DialogTitle, DialogTrigger } from "@/components/ui/dialog"
```

## File Placement

| Type | Location |
|------|----------|
| Page | `apps/web/app/<route>/page.tsx` |
| Layout | `apps/web/app/<route>/layout.tsx` |
| Shared component | `apps/web/components/<name>.tsx` |
| Feature component | `apps/web/app/<route>/_components/<name>.tsx` |
| Server action | `apps/web/app/<route>/actions.ts` |
| API route | `apps/web/app/api/<route>/route.ts` |

## Checklist Before Shipping

- [ ] Responsive: looks good on mobile, tablet, desktop
- [ ] Dark mode: toggle theme and verify
- [ ] Loading state: skeleton or suspense boundary
- [ ] Empty state: meaningful message when no data
- [ ] Error state: graceful handling with user-friendly message
- [ ] Accessibility: proper labels, keyboard navigation, contrast
- [ ] TypeScript: no `any` types, proper prop interfaces
- [ ] `pnpm typecheck` passes
