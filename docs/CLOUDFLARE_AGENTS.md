# Cloudflare Worker + Durable Object ‚Äî Agent Chat

## Overview

The agent chat can run over two transports:

1. **SSE (default)** ‚Äî `POST /api/chat` on Vercel. Subject to 60s function timeout.
2. **WebSocket (Cloudflare Worker)** ‚Äî persistent connection via Durable Object. No timeout, supports hosted tools (`web_search`, `file_search`, `code_interpreter`), and OpenAI Responses API WebSocket mode with `previous_response_id` caching.

## Tool Routing

| Route | Tools | Auth |
|-------|-------|------|
| **Gateway** | `get_parcel_details`, `screen_zoning`, `screen_flood`, `screen_soils`, `screen_wetlands`, `screen_epa`, `screen_traffic`, `screen_ldeq`, `screen_full` | `Bearer LOCAL_API_KEY` |
| **Vercel** | All other tools (deals, tasks, knowledge, calculations, etc.) | Forwarded Supabase JWT |
| **Hosted** | `web_search`, `web_search_preview`, `file_search`, `code_interpreter` | Executed by OpenAI server-side |

Gateway tools require snake_case ‚Üí camelCase argument transformation (e.g., `parcel_id` ‚Üí `parcelId`). This is handled in `tool-router.ts`.

## Auth Flow

1. Browser sends Supabase JWT as query param (`?token=<jwt>`)
2. Worker validates JWT with Supabase Auth API
3. Worker resolves org membership via `GET /api/agent/auth/resolve` on Vercel
4. JWT is stored in Durable Object state and forwarded to Vercel for tool execution

## Environment Variables

Set via `wrangler secret put <NAME>`:

| Secret | Description |
|--------|-------------|
| `OPENAI_API_KEY` | OpenAI API key for Responses API WebSocket |
| `LOCAL_API_KEY` | Gateway Bearer token |
| `LOCAL_API_URL` | Gateway base URL (e.g., `https://api.gallagherpropco.com`) |
| `SUPABASE_ANON_KEY` | Supabase anon key for JWT validation |
| `VERCEL_URL` | Vercel app URL (e.g., `https://gallagherpropco.com`) |

Set in `wrangler.toml` `[vars]`:

| Variable | Description |
|----------|-------------|
| `SUPABASE_URL` | Supabase project URL |

## DNS Setup

Add a Cloudflare Worker route or custom domain for `agents.gallagherpropco.com` pointing to the `entitlement-os-agent` Worker.

## Browser Integration

The `useAgentWebSocket` hook provides a drop-in WebSocket transport. Enable it by setting `NEXT_PUBLIC_AGENT_WS_URL` and passing `enabled: true`:

```tsx
const { sendMessage, status } = useAgentWebSocket({
  token: supabaseAccessToken,
  conversationId,
  onEvent: applyEvent,  // same handler used for SSE events
  enabled: true,
});
```

Events from the Worker match the existing `ChatStreamEvent` types, so `applyStreamingEvent` and the chat UI work unchanged.

## Conversation State

The Durable Object stores per-conversation state:

- `conversationId`, `orgId`, `userId`, `userToken`
- `lastResponseId` ‚Äî for `previous_response_id` chaining (enables context caching)
- `turnCount`, `model`, timestamps

State persists across WebSocket reconnects via DO transactional storage. After 30s of inactivity with no WebSocket, the DO may be evicted ‚Äî state is rehydrated from storage on reconnect.

## Error Handling

- **Gateway 5xx**: Retry up to 2 times with exponential backoff (1s, 2s)
- **Vercel 401/403**: Surface "session expired" error to browser
- **Tool timeout**: 30s per tool call via AbortController
- **OpenAI WebSocket close**: Reconnect once with backoff; proactive reconnect at 55min
- **OpenAI `response.failed`**: Forward error to browser, no retry

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Worker returns 401 | Check Supabase JWT validity; verify `SUPABASE_ANON_KEY` secret |
| Tool calls fail with "Unknown tool" | Re-run `npx tsx scripts/export-tools.ts` and redeploy |
| Gateway tools timeout | Check gateway health at `https://api.gallagherpropco.com/admin/health` |
| OpenAI WS won't connect | Verify `OPENAI_API_KEY` secret; check OpenAI status page |
| "No org found" from auth/resolve | User may not have an org in Prisma; check Vercel logs |
| Empty `tool-schemas.json` | The `@entitlement-os/openai` package may not be built; run `npm run build` in the monorepo first |
